#### 1.1 الهيكل التنظيمي

```
الأجزاء الداخلية:
  ├── الكيانات القانونية (Legal Entities)
  ├── المناطق / الفروع (Regions / Branches)
  ├── الأقسام (Departments)
  ├── مراكز التكلفة (Cost Centers)
  ├── مراكز الأرباح (Profit Centers)
  ├── وحدات الأعمال (Business Units)
  └── المواقع والمستودعات (Locations / Sites)

الأولوية      : CRITICAL PATH
يعتمد على     : لا شيء — نقطة البداية (ROOT)
تعتمد عليه   : جميع المكونات في المنصة بدون استثناء
ملاحظة       : مراكز التكلفة ومراكز الأرباح تُعرَّف هنا فقط —
               ترتبط بالعمليات المالية لاحقاً في 3.4

أثر Security registry:
  PERM_LEGAL_ENTITY_VIEW مُسجَّل في SecurityPermissions.java
  → يُثبت أن Organization module بدأت تتشكّل مسبقاً
    داخل الـ Security scope
  → يُؤخذ AS-IS كجزء من SEC_PAGES seeding عند بناء 1.1
```

#### 1.2 Security (يشمل إدارة المستخدمين والصلاحيات)

```
الحالة        : EXCEPTION ⚠️ — مُنفَّذ مسبقاً — AS-IS
                (UserManagement مدمج داخل Security — لا انفصال)

الأجزاء الداخلية:
  ├── نطاق الشركة     (أي شركات يراها المستخدم)
  ├── نطاق الفرع      (أي فروع يراها المستخدم)
  ├── نطاق القسم      (أي أقسام يراها المستخدم)
  ├── نطاق البيانات   (Row-Level على كل جدول)
  ├── حسابات المستخدمين (USERS)
  ├── الأدوار (ROLES)
  └── الصلاحيات (PERMISSIONS / SEC_PAGES)

الأولوية      : CRITICAL PATH
يعتمد على     : 1.1
تعتمد عليه   : جميع المكونات التي تعرض بيانات
ملاحظة       : يعرّف القواعد والمستخدمين ويطبّقهم معاً —
               لا يوجد مكوّن مستقل لإدارة المستخدمين.
               أسماء الجداول والأعمدة AS-IS ولا تتبع معايير التسمية القياسية —
               راجع master-registry Section 4 (PERMANENT EXCEPTION).

أثر Security registry (نقاط للتوثيق في P0):
  Permissions GL مسجّلة فعلاً: GL_ACCOUNT_VIEW / GL_RULE_VIEW /
  GL_JOURNAL_VIEW / GL_JOURNAL_APPROVE / GL_JOURNAL_POST /
  GL_JOURNAL_REVERSE / GL_JOURNAL_CANCEL / GL_POSTING_VIEW
  → تشير إلى أن Finance (3.4) بدأت تتشكّل على مستوى SEC_PAGES seeding
  → لا تُعدّ Finance module كـ ACTIVE — فقط permission seeds موجودة
  → تُؤخذ AS-IS كـ EXCEPTION بنفس منطق Security module نفسها
  → يُوثَّق هذا كـ INF في P0 session الخاص بـ Finance
```
